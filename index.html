TSimulus

Origins

  • EAM-SDI: research project with Guardis.
  • Specific area of ​​automatic infrastructure management.
  • Collect data of cpus, fans, ...
  • Monitoring, take reactions against events.
  • Problem: lack of real data and in real time.
  • The solution was to simulate a large number of sensors gathering information from a collection and analysis system.

TSimulus

What is TSimulus?

TSimulus (formally, RTS-Gen) is a tool for generating random, yet realistic, time series values.

Time series?
  • An ordered sequence of points in time, each of them is associated with at most one value.
  • Timestamp → value (typically, real).
  • example: 2018-03-12T14:00:00.000 - 20°C
  • Used in a wide variety of fields.

Random but realistic?

  • Convincing noise.
  • Models, patterns.

TSimulus

How TSimulus works?

In input

A DSL that describes "the look" of the timeseries

Cycles (daily, weekly)

Noise (white, ARMA)

combinations

conditions

In output

triplets (series, timestamp, value)
## TSimulus #### Documentation * Maintained on readthedocs: http://tsimulus.readthedocs.io * The library * Scala (jar downloadable from Maven Central) * Development on Github (https://github.com/cetic/TSimulus) * The CLI: https://github.com/cetic/tsimulus-cli * Reads configuration documents from a regular JSON file and produces a CSV file containing the time series. * The microservice: https://github.com/cetic/tsimulus-ms * API akka, sending a configuration file in POST request * Possible to generate a value on POST request or a set of values * CSV in response * Docker image available: https://hub.docker.com/r/ceticasbl/tsimulus-ms/
## TSAAS ~ TSimulus as a Service https://git.cetic.be/TSimulus/tsimulus-saas Work in progress at CETIC: TSimulus As A Service (TSAAS) where the project aims at * building a REST API in front of the TSimulus framework. * set of configurable websocket routes to consume the TSimulus stream. Demo environment at: https://tsaas.cetic.be/ * TSAAS emerged from Big Data research projects!
