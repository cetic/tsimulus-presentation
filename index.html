input for

Performance / Load Testing

Determines a system's performance under real-life load conditions.

Helps determine how the application behaves when there are usage spikes.

input for

Monitoring

Services and infrastructure monitoring tools provide information on the state of the system.

Should be coupled with the Load testing tests and/or with TSimulus.
(memory usage, CPU, error logs, disk space, network connectivity, ...) , to be able to monitor the different components during these tests Big data projects involve processing of large amounts of data in a short period of time. This operation requires heavy computing resources and smooth data flow in the network. Architectural issues in the system can lead to performance bottlenecks in the process, which can impact the availability of the application. This can have impact on the success of the project. Performance testing of the system is required to avoid the above issues. Here we measure metrics like throughput, memory utilization, CPU utilization, time taken to complete a task etc. It is also recommended to run failover tests to validate the fault tolerance of the system and ensure that if some nodes fail, other nodes will take up the processing.

TSimulus

What is TSimulus?

TSimulus (formally, RTS-Gen) is a tool for generating random, yet realistic, time series values.

Time series?
  • An ordered sequence of points in time, each of them is associated with at most one value.
  • Timestamp → value (typically, real).
  • example: 2018-03-12T14:00:00.000 - 20°C
  • Used in a wide variety of fields.

Random but realistic?

  • Convincing noise.
  • Models, patterns.

conditions

triplets (series, timestamp, value)
## TSimulus #### Documentation * Maintained on readthedocs: http://tsimulus.readthedocs.io * The library * Scala (jar downloadable from Maven Central) * Development on Github (https://github.com/cetic/TSimulus) * The CLI: https://github.com/cetic/tsimulus-cli * Reads configuration documents from a regular JSON file and produces a CSV file containing the time series. * The microservice: https://github.com/cetic/tsimulus-ms * API akka, sending a configuration file in POST request * Possible to generate a value on POST request or a set of values * CSV in response * Docker image available: https://hub.docker.com/r/ceticasbl/tsimulus-ms/
## TSAAS ~ TSimulus as a Service https://git.cetic.be/TSimulus/tsimulus-saas Work that is in progress at CETIC: TSimulus As A Service (TSAAS) where the project aims at * building a REST API in front of the TSimulus framework. * set of configurable websocket routes to consume the TSimulus stream. Demo environment at: https://tsaas.cetic.be/
